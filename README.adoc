= Semantic Image Search (Core)
:toc: macro
:toc-title: Table of Contents

A pure D-lang library for local, AI-powered semantic image search. This library leverages CLIP (Contrastive Language-Image Pre-training) models to enable searching through image collections using natural language descriptions, all running locally without cloud dependencies.

toc::[]

== 💡 Explanation: How it works

Traditional image search relies on filenames or metadata tags. This project uses **Semantic Embedding**:

1.  **Vision Encoder**: Pictures are converted into a "vector" (a list of numbers) that represents their visual meaning.
2.  **Text Encoder**: Your search query ("a dog playing in the snow") is converted into a vector in the same mathematical space.
3.  **Similarity Search**: We find the images whose vectors are closest to the query vector.

=== The Stack

This project is the orchestrator in a multi-layered D-lang AI stack:

[mermaid]
....
graph TD
    UI[vibe-search-desktop] --> CORE[vibe-search-core]
    CORE --> DENSOR[Densor]
    CORE --> VS[vector-store-dlang]
    DENSOR -.-> |Pure D Inference| CLIP[CLIP Model / GGUF]
    VS -.-> |Storage| DB[(Vector Index)]
....

*   **Densor**: The high-performance tensor engine (Pure D port of GGML).
*   **Vector-Store-Dlang**: A specialized database for fast vector similarity lookups.
*   **Vibe-Search-Core**: This library—handles image loading, model orchestration, and indexing logic.

== 🚀 Tutorial: Quick Start

To integrate semantic search into your D project:

1.  Add the dependency:
+
[source,bash]
----
dub add vibe-search-core
----

2.  Download a CLIP GGUF model (e.g., from HuggingFace).
3.  Initialize the indexer:
+
[source,d]
----
import vibe_search.core.indexer;
auto indexer = new ImageIndexer("./models/clip-v1.gguf");
indexer.indexDirectory("/path/to/my/photos");
----

== 🛠️ How-to: Customizing Search

To perform a search after indexing:

[source,d]
----
auto results = indexer.search("sunset over the mountains", 10);
foreach(res; results) {
    writeln("Found matching image: ", res.path, " (Score: ", res.score, ")");
}
----

== 📖 Reference: Technical Specs

*   **Inference Engine**: `densor`
*   **Vector Storage**: `vector-store-dlang`
*   **Supported Formats**: JPEG, PNG, TGA, BMP (via `imageformats`).
*   **Model Compatibility**: CLIP GGUF (v1, v2, v3).
